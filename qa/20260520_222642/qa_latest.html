Las 7 capas estándar del análisis QA

Cada análisis evalúa estas 7 capas obligatoriamente para descomponer la causa raíz del bug:

1. PlaybookEl código del playbook tal como está hoy
2. HistóricoCambios recientes al playbook (commits, PRs)
3. CatálogoDatos del Google Sheet (inventario, agent_copy)
4. OrquestadorClasificación inicial del usuario (intent, slots)
5. Backend / ToolLlamadas al petal-sheet-api
6. Política / NegocioReglas de negocio (plazos, importes, ocasiones soportadas)
7. TestCalibración del check regex del TC

Cada capa se marca con uno de los 4 estados (ver "Marca posible" para detalle).

Estados posibles de cada capa

🔴 problemaCapa comprobada con fuente. ES causa del bug.
🟢 okCapa comprobada con fuente. NO es causa del bug.
🟡 supuestaNo se pudo comprobar (falta documentación o info externa).
⚪ N/AEsta capa no aplica al tipo de bug analizado.

Las marcas 🔴 y 🟢 requieren citar la fuente entre paréntesis (ej: Read compra.yaml, git log, gh pr view, gcloud logging, curl URL).
