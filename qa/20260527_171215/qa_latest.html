Las 9 capas del análisis QA

Cada análisis evalúa estas 9 capas obligatoriamente para identificar la causa raíz del bug:

1. ComportamientoPlaybooks, Examples, Generators
2. RoutingFlows, Pages, Intents, Entity Types
3. Parámetros / SlotsPaso de slots entre playbooks y tools
4. IntegraciónTools, Webhooks, llamadas al backend
5. DatosGoogle Sheet (inventario, agent_copy, negocio)
6. InfraestructuraEnvironments, Versions, Agent Config
7. Modelo / LLMComportamiento de Gemini (alucinaciones)
8. HistóricoRegresiones — git log del playbook
9. TestCalibración del check regex del TC

Estados posibles de cada capa

🔴 problemaCapa comprobada con fuente. ES causa del bug.
🟢 okCapa comprobada con fuente. NO es causa del bug.
🟡 supuestaNo se pudo comprobar (falta documentación o info externa).
⚪ N/AEsta capa no aplica al tipo de bug analizado.

Las marcas 🔴 y 🟢 requieren citar la fuente (ej: Read compra.yaml, git log, gh pr view, gcloud logging, curl URL).
